# 青年营计分台 · 导师操作手册

你是游戏的**裁判 + 系统操作员**。所有国家的"动作"都需要导师在 dashboard 上代为执行,以保证规则一致、防止作弊。本手册涵盖:系统架构、6 个按钮的精确流程、Google Sheets 配置项、错误处理、比赛前检查清单。

---

## 1. 🎯 你的角色

| 你做什么 | 你不做什么 |
|---|---|
| 听国家请求 → 在系统操作 (采购/制造/销售/研发) | 替国家做决策 |
| 监督公平 (骰子真摇, 谈判要双方点头) | 暗中调整数据 |
| 调整 (爱心/荣誉/金币) 需附原因 | 没原因调整 (系统强制要求) |
| 见到 BUG/争议 → 立刻找主裁 | 自己改 sheet 解决 |
| 比赛后让国家看自己的日志 | 隐藏日志 |

**导师身份选择 (A/B/C/D/E)** — 每个操作都会记你的代号到 Log 表,所以**先选身份再做事**,否则按钮报错。

---

## 2. 🖥️ 系统架构 (一句话版)

```
[导师 dashboard]  ←→  [Google Apps Script]  ←→  [Google Sheets]
        ↑                                              ↑
        前端 (HTML/JS)                          所有规则/数据在 sheet
        每 3 秒拉一次数据                       可现场改
```

**关键: 几乎所有"配置项"都在 Sheets 里, 改完即时生效, 不用动代码。**

## 3. 📊 Google Sheets 表一览

| 表名 | 用途 | 你能不能改 | 改了立刻生效? |
|---|---|---|---|
| **Countries** | 国家当前状态 (金币/资源/产品计数器/level...) | 一般不直接改,用按钮 | 是 |
| **Products** | 每国库存产品 (country_id, name, qty) | 应急时手动改 | 是 |
| **Log** | 所有操作流水, 不可改 | 只看, 别改 | — |
| **Prices** | 资源/产品价格表 (price, sell_price, asset_value, price_jp) | ✅ 随时改 | 是 |
| **Config** | 全局参数 (申请费/上限/研发费/失败率/荣誉金额) | ✅ 随时改 | 是 |
| **RDPrizes** | 研发抽奖奖池 (type/value/weight/label) | ✅ 随时改 | 下次抽奖生效 |
| **LoveTable** | 爱心 → 资产倍数映射 | ✅ 随时改 | 是 |
| **LevelUp** | 升级门槛 (to_level / key / label / need) | ✅ 随时改 | 是 |
| **Recipes** | 所有产品配方 (country/level/name/资源量/semi1-4) | ✅ 随时改 | 是 |
| **InitialState** | 重置时各国回到的初始值 | ✅ 比赛前设定好 | 下次重置生效 |
| **PurchaseOrders** | 采购单流水 (active/consumed) | 应急时手动改 | 是 |
| **CountryTokens** | 国家只读链接 token | 改后告诉国家新链接 | 是 |

> ⚠️ **改 Prices.price 后, 已经在采购单中的产品不受影响 (扣金币时已经按当时价算了),只对新订单生效。**

## 4. 🎮 6 个按钮详解 (每国卡片)

按钮位置:**金币卡片下方, 2 行 × 3 列**。点哪国卡片就操作哪国。

每个动作用统一格式: ① **📋 条件清单** ② **🎲 机制 / 骰子规则** (如有) ③ **🎯 例子** (国家说 X → 你做 Y) ④ **📊 前后对比 / 注意要点**。

### 4.1 🛒 购买 — 国家用金币向银行买基础资源 / L1 / L2 / L3

#### 📋 条件 (全部满足后端才扣钱)

| # | 条件 | 说明 |
|---|---|---|
| 1 | 💰 国家金币足够 | 单价 × 数量 ≤ 当前金币 |
| 2 | 🔑 国家等级解锁该 tab | 资源 Lv.1+ · L1 Lv.2+ · L2 Lv.3+ · L3 Lv.4 |
| 3 | 🌐 产品归属合规 | L1 仅本国; L2/L3 任意国家; L4 不出售 |
| 4 | 📝 你填了购买原因 | 后端强制 |

#### 🎯 Tab 说明 (产品归属规则)

| Tab | 谁能买 | 用途 |
|---|---|---|
| 基础资源 | 任何国家 (Lv.1+) | 6 种原料 |
| L1 产品 | 仅本国 (Lv.2+) | L1 由各国自产, 不开放跨国 |
| L2 产品 | 任意国家 (Lv.3+) | 跨国买高级半成品凑配方 |
| L3 产品 | 任意国家 (Lv.4) | 跨国买 L3 凑 L4 配方 |
| ~~L4 产品~~ | 无 | 银行不卖, 跨国也不卖 |

锁住的 tab 显示为 `🔒 LX 产品` (淡红色), 点不动; 后端也会拒。

#### 🎯 例子: 国家 = MY · "我要买 200 单位 oil"

| # | 操作 | 结果 |
|---|---|---|
| 1 | 你点 **🛒 购买** → 选 Tab "基础资源" | 列出 6 种资源 |
| 2 | 点 🛢 石油 | 显示单价 500/100 单位 |
| 3 | 数量步进 → 2 块 (2×100=200 单位) | 总价 1,000 金币 |
| 4 | 填原因 → 确认 | 金币 −1,000 · 石油 +200 单位 |

#### ⚠️ 注意

- 🇯🇵 日本买资源自动用 `Prices.price_jp` (=450), 列表显示红色 "JP" 角标
- L2 / L3 列表每行带来源国家 chip (🇲🇾 / 🇰🇷 / 🇯🇵 / 🇺🇸 或 "本国")
- 银行价固定 (Prices.price); 国家想砍价/抬价 → 走 💸 销售 modal 双方谈

### 4.2 📋 订单 — 国家花金币换一张采购单 (生产前提)

#### 📋 条件

| # | 条件 | 说明 |
|---|---|---|
| 1 | 💰 足够金币 | 每张 `Config.purchase_cost` (默认 100) |
| 2 | 📦 持单未满 | Lv.1→2 · Lv.2→3 · Lv.3→5 · Lv.4→5 张 (`purchase_limit_l1..l4`) |
| 3 | 🔑 申请级别 ≤ 国家等级 | 高于国家等级的按钮显示 🔒, 前后端双重拒绝 |
| 4 | 📦 本国在该 Level 有产品池 | Recipes 表有该国 LX 行 |

#### 🎲 抽奖机制

| 候选 | 权重 |
|---|---|
| 国家**已持有**的产品 | 1 |
| 国家**没持有**的产品 | 5 (`Config.purchase_dup_ratio`, 可改) |

→ 想凑齐不同产品比重复抽快 5 倍, 仍可能重复

#### 🎯 例子: 国家 = MY Lv.2 · "我要申请 L1 采购单"

| # | 操作 | 结果 |
|---|---|---|
| 1 | 你点 **📋 订单** | 顶部显示 "Lv.2 · 持单 1/3 · 金币 18,200" |
| 2 | 点 **L1** 大按钮 (L3/L4 显示 🔒) | 系统扣 100 金币 |
| 3 | 系统按权重随机抽 | toast: "采购单 L1: 石油燃料包" |
| 4 | 卡片刷新 | 持单 2/3 · 金币 18,100 |

#### ⚠️ 注意

- 申请前后端**实时读 Config**, 改 sheet 立刻生效
- 系统从该国 RECIPES 池**随机**抽 → 国家不能挑产品
- 满了 → 国家先去 🏭 生产消耗掉, 或升级国家

### 4.3 🏭 生产 — 国家把资源 + 半成品按配方做成产品

#### 📋 条件

| # | 条件 | 说明 |
|---|---|---|
| 1 | 📋 持有该产品的 active 采购单 | 没单 → 制造列表看不到 |
| 2 | 💧 资源够 | 按 Recipe 各项 (modal 里缺料显红) |
| 3 | ⚙️ 半成品够 | L2+ 需要 semi1-4 库存全达标 |
| 4 | 📝 你填了生产原因 + 输入骰子 (1-6) | 后端强制 |

#### 🎲 骰子机制

| 骰点 | 结果 | 影响 |
|---|---|---|
| 🎲 **1** (17%) | 💥 失败 | 资源扣 · 采购单作废 · **不出产品** (计数器 `lN_orders` 仍 +1) |
| 🎲 **2-6** (83%) | ✅ 成功 | 资源扣 · 采购单作废 · 产品库存 **+1** · 计数器 +1 |

#### 🎯 例子: 国家 = KR · "我要做 AI电脑" (L2)

| # | 操作 | 结果 |
|---|---|---|
| 1 | 国家说要做 AI电脑 | 必须先有 AI电脑 的采购单 |
| 2 | 你点 **🏭 生产** → 列表里点 AI电脑 | 系统显示配方:100 metal + 200 elec + 200 chips + 1 基础电脑系统 |
| 3 | 让国家**亲手丢透明骰** | 🎲 3 |
| 4 | 你按数字 3 | (4 号默认红, 别按错) |
| 5 | 填原因 → 确认 | 资源/半成品扣 · 采购单作废 · AI电脑 +1 · `l2_orders` +1 |
| 6 | 全屏动画 | ✅ 成功 |

#### ⚠️ 注意

- L3/L4 同样规则 — 都要丢骰
- **失败时别帮国家挽回** (违反规则)
- 点不出某产品 → 检查是不是没采购单

### 4.4 💸 销售 — 国家把成品卖给银行或其他国家

#### 📋 条件

| # | 条件 | 说明 |
|---|---|---|
| 1 | 📦 库存足够 | 产品数 ≥ 销售数量 |
| 2 | 🎯 选定对象 | 🏦 银行 (固定 sell_price) **或** 🇲🇾/🇰🇷/🇯🇵/🇺🇸 (协商价) |
| 3 | 🚫 不能卖给自己 | 后端拒 |
| 4 | 🔒 L4 只能卖银行 | L4 跨国后端拒 + 前端 🔒 |
| 5 | 📝 填销售原因 | 后端强制 |
| 6 | 🎲 输入运输骰 *或* 🎫 用免失败卡 | 二选一 |

#### 🎲 运输骰机制

| 骰点 | 结果 | 影响 |
|---|---|---|
| 🎲 **4** (17%) | 💥 运输失败 | 产品扣 · **金币不加** · `shipments` 不加 |
| 🎲 **1/2/3/5/6** (83%) | ✅ 成功 | 产品扣 · 金币 +(qty × 单价) · `shipments` +1 |
| 🎫 **免失败运输卡** | ✅ 自动成功 (100%) | 跳过骰子 · 卡 −1 |

#### 🎯 例子: 国家 = MY · "我要卖 3 个石油燃料包给银行"

| # | 操作 | 结果 |
|---|---|---|
| 1 | 你点 **💸 销售** | 库存列表显示 "石油燃料包 ×5 · 银行价 2,000" |
| 2 | 点它 → 选销售对象 🏦 银行 | 单价自动锁定 2,000 |
| 3 | 数量 → 3 | 总额 6,000 金币 |
| 4 | 国家丢实体运输骰 → 🎲 **3** | 你按 3 (4 号会高亮红色提醒) |
| 5 | 填原因 → 确认 | 产品 -3 · 金币 +6,000 · shipments +1 |

#### ⚠️ 注意

- 跨国协商价完全由两国谈, 你只输入数字, 不否决 (除非违反明显规则)
- 卖给银行用 `Prices.sell_price` — **KR L1 的 +15% 加成** 写死在 sell_price=2,300 (其他国家 L1=2,000)
- **JP 没有销售加成** (只有买便宜)
- 🔒 **L4 不能跨国销售** — L4 产品的国家 tab 显示 🔒, 后端 `sell_` 拒绝 `to !== 'bank'`

### 4.5 🎁 研发 (R&D) — 国家花金币随机抽奖

#### 📋 条件

| # | 条件 | 说明 |
|---|---|---|
| 1 | 💰 足够金币 | 每次 `Config.rd_cost` (默认 500) |
| 2 | 📝 填原因 | 默认填好 "研发部投资" |
| 3 | 🎁 奖池非空 | RDPrizes 表至少一行 |

#### 🎲 机制

| 阶段 | 概率 | 结果 |
|---|---|---|
| 立刻扣费 | 100% | 金币 −`rd_cost` (无论后续) |
| 💥 失败 | `Config.rd_fail_rate` (默认 30%) | 钱没了, 啥也不拿 |
| ✅ 成功 | 1 − rd_fail_rate | 从奖池按 weight 抽一项 |

#### 🎁 奖池 (RDPrizes 表)

可改; 每行: `type` / `value` / `weight` / `label` / `note`。常见类型:

| type | 效果 |
|---|---|
| `res` | 随机一种资源 +value 单位 |
| `nofail` | 免失败运输卡 ×value |
| `coins` | 金币 +value |

#### 🎯 例子: 国家 = US · "我要研发"

| # | 操作 | 结果 |
|---|---|---|
| 1 | 你点 **🎁 研发** | modal 显示 500 金币 · 失败率 30% · 奖池预览 |
| 2 | 国家确认 → 你点 "投资研发" | 后端立刻扣 500 |
| 3 | 后端随机 | 30% → 💥 失败 / 70% → 抽奖 |
| 4 | 全屏动画 | 🎉 成功 (举例抽中 "金币 +2,000") 或 💥 失败 |

#### ⚠️ 注意

- **失败钱不退** — 别心软
- 研发费在 `Config.rd_cost`, 失败率在 `Config.rd_fail_rate`, 改 sheet 立即生效
- 奖品权重 (weight) 决定命中率, 别让某项权重 ×10 失衡

## 6. ✋ 调整 (Adjust) — 给国家手动加扣金币 / 爱心 / 荣誉

#### 📋 条件

| # | 条件 | 说明 |
|---|---|---|
| 1 | 👤 已选导师身份 (A/B/C/D/E) | 后端拒未选导师的调整 |
| 2 | 📝 必填原因 | 后端强制非空 |
| 3 | 🔢 delta 非 0 | 必须是数字 |
| 4 | 🔑 `level` 字段上限 4 | 后端拒 `level > 4` (没有 Lv.5) |
| 5 | ✅ field 在允许列表 | coins / love / honor / level / 各资源 / 计数器等 |

#### 🎯 流程

| # | 操作 | 结果 |
|---|---|---|
| 1 | 点国家卡片的**数字本身** (金币 60,145 那串大字) | 弹出调整 modal |
| 2 | 输入正数加 / 负数减 (例 +500 或 -200) | preview 显示新值 |
| 3 | **必填原因** (具体一点) | 例 "完成图书馆任务 (5/5 题)" |
| 4 | 确认 | 字段更新 + 一条 `adjust` log |

#### 🎯 调什么时候

| 场景 | 调什么 | 量 |
|---|---|---|
| 完成挑战任务 | 💰 金币 +X | 视任务难度 |
| 助人为乐 / 团队精神 | ❤️ 爱心 +1 | 适度, 别滥发 |
| 优秀代表 / 演讲精彩 | 🏅 荣誉 +1 | 适度 |
| 违规 (作弊 / 冲突) | 💰 金币 −X / ❤️ 爱心 −1 | 警告制 |
| 协助其他国家 | ❤️ 爱心 +1 | 协作时 |

#### 📝 原因书写规则

| | 例子 |
|---|---|
| ❌ 太短, 看不出为啥 | "奖励" |
| ✅ 具体, 可审计 | "完成图书馆任务 (5/5 题)" |
| ✅ 有据可查 | "无故缺席集合, 扣 200" |

所有调整都进 Log, 比赛后可以查谁加扣过什么。

## 7. ⚙️ Config 表常用调整

随时改 sheet 单元格, 不用重新部署:

| key | 当前值 | 含义 |
|---|---|---|
| `purchase_cost` | 100 | 申请一张采购单的金币成本 |
| `rd_cost` | 500 | 研发部投资的金币成本 |
| `rd_fail_rate` | 0.3 | 研发失败概率 (0-1), 失败时金币不退 |
| `jp_res_price` | 450 | 日本购买基础资源的统一单价 (留空则回退到 price 列) |

| `purchase_limit_l1` | 2 | 国家 Lv.1 时同时持有最大采购单数量 |
| `purchase_limit_l2` | 3 | 国家 Lv.2 时同时持有最大采购单数量 |
| `purchase_limit_l3` | 5 | 国家 Lv.3 时同时持有最大采购单数量 |
| `purchase_limit_l4` | 5 | 国家 Lv.4 时同时持有最大采购单数量 |

## 8. 📐 LoveTable (爱心倍数)

公式: `总资产 = ((金币 + 产品估值) × 爱心倍数) + (荣誉值 × 1000)`

默认表:
| ❤️ 爱心 | 倍数 | 说明 |
|---|---|---|
| 0 | 1.00 | 基础 · 无加成 |
| 1 | 1.10 | +10% |
| 2 | 1.12 | +12% |
| 3 | 1.15 | +15% |
| 4 | 1.18 | +18% |
| 5 | 1.20 | +20% |
| 6 | 1.22 | +22% |
| 7 | 1.25 | +25% |
| 8 | 1.28 | +28% |
| 9 | 1.29 | +29% |
| 10 | 1.30 | +30% |
| 11 | 1.31 | +31% |
| 12 | 1.32 | +32% |
| 13 | 1.33 | +33% |
| 14 | 1.34 | +34% |
| 15 | 1.35 | +35% |

**阶梯函数:** love=4 没行, 用 love=3 的 1.15。love=10 超过最高, 锁在 1.20。

**想加难度?** 加新行 love=10, multiplier=1.30 → 5-9 之间还是 1.20。

## 8a. 📈 LevelUp (升级门槛 + 🏆 终极完成目标)

每行 = 一个条件; 多行同一个 `to_level` 是 AND (全部达标才能升)。

> **关于 `to_level=5`:** 这些行是 **Lv.4 终极完成目标** (赢游戏的最终条件), **不是** Lv.5 升级。`level` 字段最大为 4 (后端 `adjust_` 强制); 即便条件全达成, level 也不会变 5, 只是国家卡片显示「🏆 终极完成 ✓」徽章。

| to_level | key | label | need | note |
|---|---|---|---|---|
| 2 | l1_orders | 完成 L1 产品 | 10 |  |
| 2 | coins | 金币 | 20000 |  |
| 2 | shipments | 成功运输 | 5 |  |
| 3 | l2_orders | 完成 L2 产品 | 8 |  |
| 3 | coins | 金币 | 50000 |  |
| 4 | l3_orders | 完成 L3 产品 | 5 |  |
| 4 | coins | 金币 | 90000 |  |
| 5 | l4_distinct | 不同 L4 产品 | 2 | 需做出 2 个不同 L4 产品 |
| 5 | coins | 金币 | 120000 |  |
| 5 | love | 爱心值 | 1 |  |
| 5 | honor | 荣誉值 | 30 |  |

**支持的 key** (匹配 Countries 表的列, 或派生值):
- `coins`, `love`, `honor`
- `l1_orders`, `l2_orders`, `l3_orders`, `l4_orders` (累积尝试次数, 含失败)
- `shipments` (成功运输次数)
- `l4_distinct` (派生: 已生产过的不同 L4 产品种类数)

**编辑技巧:**
- 想让某级简单 → 改 need 值
- 想多加一个条件 → 新增一行同 to_level
- 想去掉一个条件 → 删那一行
- 想增加 Lv.5? → 前端会忽略 to_level > 4 的行 (硬编码上限),需要前端 + 后端配套放开 `Math.min(4, ...)` clamp

改完 sheet **立刻生效** (不用重新部署),所有国家卡片上的升级进度条会在下次刷新时按新规则显示。

## 8b. 🧪 Recipes (产品配方)

每行 = 一个产品。共 14 列:

| country | level | name | water | oil | wood | metal | electricity | chips | semi1 | semi2 | semi3 | semi4 | note |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| my | 1 | 石油燃料包 | 100 | 100 | | | | | | | | | |
| my | 2 | 工业燃料系统 | | 300 | | | 100 | | 基础发电机 | 石油燃料包 | | | |
| jp | 2 | 实验仪器 | | | | 200 | | 300 | 防护装备 | | | | |
| us | 3 | 太空科技 | | | | 200 | 200 | 200 | 投资系统 | 战略无人机 | 紧急救援系统 | | |

**怎么编辑:**
- 改资源量 → 改那 6 个资源列里的数字 (空白=不需要)
- 改半成品 → 改 semi1/2/3/4 那 4 列里的产品名字 (空白=不需要)
- 加新产品 → 新增一行,填好 country/level/name 和需要的资源/semi
- 改产品名 → 改 name 列,**注意:** 现有库存 / 其他配方里的 semi 引用不会自动更名,需要手动同步

**支持的 country 值:** `my`, `kr`, `jp`, `us` (小写)
**支持的 level 值:** 1, 2, 3, 4

**陷阱:**
- semi 名字要**精确匹配** name 列 (注意空格、繁简体)
- 同名产品只能有一行 (如果重复,后面的覆盖前面的)
- L3/L4 产品可以引用其他国家的 L2 半成品 (如 KR AI核心系统 用了 MY 的 工业燃料系统)

改完 sheet **立刻生效** (前端下次刷新即用新配方)。

## 9. 🔥 常见问题 (FAQ)

### Q: 国家说"我刚刚明明 +200 资源, 数字怎么没变?"
A: 看右下角 sync 状态 — 应该是 ✅ 已同步。还没变就硬刷 (Ctrl+Shift+R)。如果还是没变, 看 Log 表最近一条, 确认操作真的成功了。

### Q: 国家说"我有采购单, 怎么制造按钮里没显示?"
A: 三种可能:
1. 采购单的 `status` 不是 `active` (看 PurchaseOrders 表)
2. 产品名字跟 RECIPES 不对应 (运 Apps Script 里的 `repairProductNames()`)
3. 缓存 → 硬刷

### Q: 销售失败了 (骰=4), 国家说"再让我丢一次"
A: **不行**。骰子结果只算第一次。如果允许重丢, 公平性崩溃。
**例外:** 如果**导师按错按钮** (4 按成 5), 找主裁修正,在 Log 加 reason "修正 5→4"。

### Q: 两国谈了一笔买卖, 谈完反悔
A: 看是不是已经按了"确认销售":
- 没按 → 关 modal, 没事
- 按了 → 已经入 Log, 不能直接撤。让两国协商**反向再做一次** (相同价回卖)

### Q: 国家说"我们 4 个一起协作做事, 给所有人加爱心"
A: 你可以给**每个国家**点卡片上的爱心 +1, 原因写一样。系统不支持批量,每国分别加一次。

### Q: 比赛途中 dashboard 卡了 / 打不开
A:
- 检查 WiFi
- 看 `WEBAPP_URL` 部署状态 (Apps Script → 部署 → 管理部署 → 当前版本)
- 实在不行 → 启用应急模式: 用 Google Sheet 直接看 Countries 表里的数, 手动记录

### Q: 国家拿到一张研发**免失败卡**, 却不知道怎么用
A: 卡显示在国家卡顶部右上 🎫×N。**销售 modal** 才能用 (生产现在不能用, 已改成"研发是奖品, 运输才用卡")。

## 11. 🆘 紧急情况

### "我误删了一行 Products / Countries 数据"
- Google Sheet 顶部 → 编辑 → 撤销 (Ctrl+Z)
- 或 文件 → 版本历史 → 恢复到几分钟前

### "整个 dashboard 卡死"
1. 浏览器关掉重开
2. 还卡 → 用其他浏览器 (Chrome / Edge)
3. 还卡 → 主裁电脑用大屏直接登 Apps Script 看 Sheets

### "Apps Script 报错 'Exceeded maximum execution time'"
- 单次操作太慢, 通常是 Sheet 行数太多
- 让国家先停 1 分钟, 让 Apps Script 缓存清空
- 还不行 → `reset()` (征求所有国家同意!)

### "我手抖把日志清了"
- Google Sheet 版本历史恢复
- 真没了 → 老实告诉国家"日志重建中, 不影响比赛"
