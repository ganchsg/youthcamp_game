# 青年营计分台 · 导师操作手册

你是游戏的**裁判 + 系统操作员**。所有国家的"动作"都需要导师在 dashboard 上代为执行,以保证规则一致、防止作弊。本手册涵盖:系统架构、6 个按钮的精确流程、Google Sheets 配置项、错误处理、比赛前检查清单。

---

## 1. 🎯 你的角色

| 你做什么 | 你不做什么 |
|---|---|
| 听国家请求 → 在系统操作 (采购/制造/销售/研发) | 替国家做决策 |
| 监督公平 (骰子真摇, 谈判要双方点头) | 暗中调整数据 |
| 调整 (爱心/荣誉/金币) 需附原因 | 没原因调整 (系统强制要求) |
| 见到 BUG/争议 → 立刻找主裁 | 自己改 sheet 解决 |
| 比赛后让国家看自己的日志 | 隐藏日志 |

**导师身份选择 (A/B/C/D/E)** — 每个操作都会记你的代号到 Log 表,所以**先选身份再做事**,否则按钮报错。

---

## 2. 🖥️ 系统架构 (一句话版)

```
[导师 dashboard]  ←→  [Google Apps Script]  ←→  [Google Sheets]
        ↑                                              ↑
        前端 (HTML/JS)                          所有规则/数据在 sheet
        每 3 秒拉一次数据                       可现场改
```

**关键: 几乎所有"配置项"都在 Sheets 里, 改完即时生效, 不用动代码。**

## 3. 📊 Google Sheets 表一览

| 表名 | 用途 | 你能不能改 | 改了立刻生效? |
|---|---|---|---|
| **Countries** | 国家当前状态 (金币/资源/产品计数器/level...) | 一般不直接改,用按钮 | 是 |
| **Products** | 每国库存产品 (country_id, name, qty) | 应急时手动改 | 是 |
| **Log** | 所有操作流水, 不可改 | 只看, 别改 | — |
| **Prices** | 资源/产品价格表 (price, sell_price, asset_value, price_jp) | ✅ 随时改 | 是 |
| **Config** | 全局参数 (申请费/上限/研发费/失败率/荣誉金额) | ✅ 随时改 | 是 |
| **RDPrizes** | 研发抽奖奖池 (type/value/weight/label) | ✅ 随时改 | 下次抽奖生效 |
| **LoveTable** | 爱心 → 资产倍数映射 | ✅ 随时改 | 是 |
| **LevelUp** | 升级门槛 (to_level / key / label / need) | ✅ 随时改 | 是 |
| **InitialState** | 重置时各国回到的初始值 | ✅ 比赛前设定好 | 下次重置生效 |
| **PurchaseOrders** | 采购单流水 (active/consumed) | 应急时手动改 | 是 |
| **CountryTokens** | 国家只读链接 token | 改后告诉国家新链接 | 是 |

> ⚠️ **改 Prices.price 后, 已经在采购单中的产品不受影响 (扣金币时已经按当时价算了),只对新订单生效。**

## 4. 🎮 6 个按钮详解 (每国卡片)

按钮位置:**金币卡片下方, 2 行 × 3 列**。点哪国卡片就操作哪国。

### 🛒 购买 (银行采购)

**用途:** 国家用金币向银行买基础资源 / L1 / L2 产品

**流程:**
1. 国家说"我要买 200 单位 oil"
2. 你点 **🛒 购买** → 选 Tab "基础资源" → 点 oil
3. 数量步进 (每步 1 = 100 单位, 因为 unit_size=100) → 输入原因
4. 系统检查金币够不够 → 扣钱 → 加资源 → 写 Log
5. 关 modal, 看卡片资源数变化

**注意:**
- 日本买资源会自动用 `price_jp` (=450), 列表会显示红色 "JP" 角标
- L1/L2 只能买**本国**产品 (因为是当半成品自用)
- 跨国买半成品要走 💸 销售 (国家间贸易)

---

### 📋 订单 (申请采购单)

**用途:** 国家花金币换一张采购单,然后才能生产

**流程:**
1. 国家说"我要申请 L2 采购单"
2. 你点 **📋 订单** → 看顶部状态 "持单 X/2 · 金币 XXX"
3. 点 L1/L2/L3/L4 大按钮 → 系统扣 100 金币 → 随机抽该级别产品 → 显示 toast "采购单 L2: AI电脑"
4. 下方"当前持有"列表会立刻刷新

**注意:**
- **每张 100 金币** (在 Config.purchase_cost, 可改)
- **最多 2 张同时持有** (在 Config.purchase_limit, 可改)
- 申请前后端**实时读 Config**, 改 sheet 立刻生效
- 系统会从该国的 RECIPES 池里**随机**抽 → 国家不能挑

---

### 🏭 生产

**用途:** 国家把资源 + 半成品按配方做成产品

**流程:**
1. 国家说"我要做 AI电脑" (前提是有该订单)
2. 你点 **🏭 生产** → 看到列表里有 "📋 AI电脑" → 点它
3. 系统显示配方 + 是否够料 (红色 = 缺料)
4. 让国家**亲手丢实体骰子**(透明骰子,大家都看得到!)
5. 你按下骰子结果数字 (1-6)
6. 输入原因 (默认填好) → 确认
7. 全屏弹出 ✅ 成功 或 💥 失败动画

**骰规则:**
- 🎲 **1 = 失败** (资源照扣, 采购单作废, 不出产品)
- 🎲 **2-6 = 成功** (出产品 + 采购单作废)

**注意:**
- L3/L4 也是同样规则 — 都要丢骰
- 失败时**别帮国家挽回** (违反规则)
- 如果点不出某产品 → 检查是不是没采购单

---

### 💸 销售

**用途:** 国家把成品卖给银行 (固定价) 或其他国家 (谈判价)

**流程:**
1. 国家说"我要卖 3 个石油燃料包给银行"
2. 你点 **💸 销售** → 库存列表显示"石油燃料包 ×4 · 银行价 1200"
3. 点它 → 选销售对象 (🏦 银行 / 🇰🇷 KR / 🇯🇵 JP / 🇺🇸 US)
4. 输数量, 银行价格自动锁定, 国家间需双方谈好协商价
5. **国家**丢实体运输骰
6. 你按运输骰按钮 (4 号会**默认红色**, 一眼可见)
7. 输入原因 → 确认

**骰规则:**
- 🎲 **4 = 运输失败** (产品照扣, 但没金币, shipments 不加)
- 🎲 **1/2/3/5/6 = 成功** (产品扣, 金币加, shipments +1)

**🎫 免失败运输卡:**
- 如果国家持有 🎫 卡, 销售 modal 会出现"使用免失败运输卡"勾选框
- 勾上 → 跳过骰子 → 自动成功 → 卡 -1
- 国家可以**自己选择**用不用

**注意:**
- 国家间**协商价**完全由两国谈, 你只输入数字, 不能否决 (除非违反明显规则)
- 卖给银行用的是 Prices.sell_price (含 KR L1 的 +10% 加成,已写死在 KR L1 那几行的 sell_price)
- **JP 没有特殊销售加成** (只有买便宜)

---

### 🎁 研发 (R&D 投资)

**用途:** 国家花 500 金币随机抽奖

**流程:**
1. 国家说"我要研发"
2. 你点 **🎁 研发** → 看到费用 500, 失败率 20%, 奖品池预览
3. 国家确认 → 你点"投资研发"
4. 后端: 立刻扣 500 → 掷骰 (random) → 20% 失败 / 80% 抽奖品池
5. 全屏显示结果 (🎉 成功 / 💥 失败)

**奖品池** (RDPrizes 表, 可改):
- 资源包 ×2 (随机一种资源 +200)
- 免失败运输卡 ×1 / ×2
- 金币 +1000 / +2000

**注意:**
- **失败钱不退** — 这是规则, 别心软
- 研发费在 Config.rd_cost (默认 500), 失败率在 Config.rd_fail_rate (默认 0.2)
- 奖品权重 (weight 列) 决定中奖概率, 别让某项权重 ×10 让游戏失衡

---

### 📜 日志

**用途:** 查看该国所有操作历史

- 最近 100 条 (按时间倒序)
- 顶部 Tab 可过滤: 调整 / 采购单 / 采购 / 制造 / 失败 / 销售 / 运输失败 / 研发 / 研发失败
- 国家在比赛后**有权**看自己的日志, 你给他们看
- 全局日志: 顶部 📜 按钮 (看所有国家)

## 5. 🔄 重置游戏 (按钮在顶部)

**何时用:** 上一场比赛结束, 准备下一场

**流程:**
1. 顶部右上 🔄 按钮 (红色, 只有导师能看到)
2. **Step 1** — 弹窗列出会清的东西 (日志/产品/订单/资源/卡/计数器, 金币/资源回 InitialState, 爱心/荣誉/level 归零)
3. 点"我看过了 → 下一步"
4. **Step 2** — 必须打 `RESET` (大写敏感) 才能解锁红色"确认重置"
5. 按下 → 后端清表 + 写一条 `RESET` log

**重置后保留的:** Prices / Config / RDPrizes / LoveTable / InitialState / CountryTokens
**重置后清掉的:** Log, Products, PurchaseOrders, 国家所有 dynamic state

> ⚠️ **比赛中绝对不要按重置。** 这是上下场之间用的。

## 6. ✋ 调整 (Adjust)

**用途:** 国家因为表现优秀/违规, 你给加 / 扣金币 / 爱心 / 荣誉

**怎么调:**
- 点国家卡片上的**数字**本身 (金币 60145 那串大字)
- 弹窗 → 输入正数加 / 负数减 → **必须填原因**
- 例: 金币 +500 原因 "完成任务奖励 (清洁活动)"

**调什么时候?**

| 场景 | 调什么 | 范围 |
|---|---|---|
| 完成挑战任务 | 金币 +X | 视任务难度 |
| 助人为乐/团队精神 | 爱心 +1 | 适度,别滥发 |
| 优秀代表/演讲精彩 | 荣誉 +1 | 适度 |
| 违规 (作弊/冲突) | 金币 -X / 爱心 -1 | 警告制 |
| 协助其他国家 | 爱心 +1 | 协作时 |

**原因填什么:**
- ❌ "奖励" — 太短, 看不出为啥
- ✅ "完成图书馆任务 (5/5 题)" — 具体
- ✅ "无故缺席集合, 扣 200" — 有据可查

所有调整都进 Log, 比赛后可以查谁加扣过什么。

## 7. ⚙️ Config 表常用调整

随时改 sheet 单元格, 不用重新部署:

| key | 默认 | 含义 | 何时改 |
|---|---|---|---|
| `purchase_cost` | 100 | 申请一张采购单的金币 | 觉得太便宜或太贵 |
| `purchase_limit` | 2 | 同时持有最多几张采购单 | 想加快/放慢节奏 |
| `rd_cost` | 500 | 研发投资金币成本 | 想让研发更容易/更难 |
| `rd_fail_rate` | 0.2 | 研发失败概率 (0-1) | 想调奖励难度 |
| `jp_res_price` | 450 | 日本买资源单价 (仅展示, 实际价取自 Prices.price_jp) | 别动 |
| `honor_coin` | 1000 | 每点荣誉 = 多少金币加进资产 | 想让荣誉更值钱 |

## 8. 📐 LoveTable (爱心倍数)

公式: `总资产 = ((金币 + 产品估值) × 爱心倍数) + (荣誉值 × 1000)`

默认表:
| love | multiplier |
|---|---|
| 0 | 1.00 |
| 1 | 1.10 |
| 2 | 1.12 |
| 3 | 1.15 |
| 5 | 1.20 |

**阶梯函数:** love=4 没行, 用 love=3 的 1.15。love=10 超过最高, 锁在 1.20。

**想加难度?** 加新行 love=10, multiplier=1.30 → 5-9 之间还是 1.20。

## 8a. 📈 LevelUp (升级门槛)

每行 = 一个条件; 多行同一个 `to_level` 是 AND (全部达标才能升)。

| to_level | key | label | need | note |
|---|---|---|---|---|
| 2 | l1_orders | 完成 L1 产品 | 10 | |
| 2 | coins | 金币 | 20000 | |
| 2 | shipments | 成功运输 | 5 | |
| 3 | l2_orders | 完成 L2 产品 | 5 | |
| 3 | coins | 金币 | 50000 | |
| 4 | l3_orders | 完成 L3 产品 | 3 | |
| 4 | coins | 金币 | 80000 | |
| 5 | l4_distinct | 不同 L4 产品 | 2 | |
| 5 | coins | 金币 | 80000 | |
| 5 | love | 爱心值 | 1 | |
| 5 | honor | 荣誉值 | 1 | |

**支持的 key** (匹配 Countries 表的列, 或派生值):
- `coins`, `love`, `honor`
- `l1_orders`, `l2_orders`, `l3_orders`, `l4_orders` (累积尝试次数, 含失败)
- `shipments` (成功运输次数)
- `l4_distinct` (派生: 已生产过的不同 L4 产品种类数)

**编辑技巧:**
- 想让某级简单 → 改 need 值
- 想多加一个条件 → 新增一行同 to_level
- 想去掉一个条件 → 删那一行
- 想增加 Lv.6? → 新增 to_level=6 的行 (但前端目前只支持到 Lv.5, 需要前端配套改)

改完 sheet **立刻生效** (不用重新部署),所有国家卡片上的升级进度条会在下次刷新时按新规则显示。

## 9. 🔥 常见问题 (FAQ)

### Q: 国家说"我刚刚明明 +200 资源, 数字怎么没变?"
A: 看右下角 sync 状态 — 应该是 ✅ 已同步。还没变就硬刷 (Ctrl+Shift+R)。如果还是没变, 看 Log 表最近一条, 确认操作真的成功了。

### Q: 国家说"我有采购单, 怎么制造按钮里没显示?"
A: 三种可能:
1. 采购单的 `status` 不是 `active` (看 PurchaseOrders 表)
2. 产品名字跟 RECIPES 不对应 (运 Apps Script 里的 `repairProductNames()`)
3. 缓存 → 硬刷

### Q: 销售失败了 (骰=4), 国家说"再让我丢一次"
A: **不行**。骰子结果只算第一次。如果允许重丢, 公平性崩溃。
**例外:** 如果**导师按错按钮** (4 按成 5), 找主裁修正,在 Log 加 reason "修正 5→4"。

### Q: 两国谈了一笔买卖, 谈完反悔
A: 看是不是已经按了"确认销售":
- 没按 → 关 modal, 没事
- 按了 → 已经入 Log, 不能直接撤。让两国协商**反向再做一次** (相同价回卖)

### Q: 国家说"我们 4 个一起协作做事, 给所有人加爱心"
A: 你可以给**每个国家**点卡片上的爱心 +1, 原因写一样。系统不支持批量,每国分别加一次。

### Q: 比赛途中 dashboard 卡了 / 打不开
A:
- 检查 WiFi
- 看 `WEBAPP_URL` 部署状态 (Apps Script → 部署 → 管理部署 → 当前版本)
- 实在不行 → 启用应急模式: 用 Google Sheet 直接看 Countries 表里的数, 手动记录

### Q: 国家拿到一张研发**免失败卡**, 却不知道怎么用
A: 卡显示在国家卡顶部右上 🎫×N。**销售 modal** 才能用 (生产现在不能用, 已改成"研发是奖品, 运输才用卡")。

### 一周前

## 11. 🆘 紧急情况

### "我误删了一行 Products / Countries 数据"
- Google Sheet 顶部 → 编辑 → 撤销 (Ctrl+Z)
- 或 文件 → 版本历史 → 恢复到几分钟前

### "整个 dashboard 卡死"
1. 浏览器关掉重开
2. 还卡 → 用其他浏览器 (Chrome / Edge)
3. 还卡 → 主裁电脑用大屏直接登 Apps Script 看 Sheets

### "Apps Script 报错 'Exceeded maximum execution time'"
- 单次操作太慢, 通常是 Sheet 行数太多
- 让国家先停 1 分钟, 让 Apps Script 缓存清空
- 还不行 → `reset()` (征求所有国家同意!)

### "我手抖把日志清了"
- Google Sheet 版本历史恢复
- 真没了 → 老实告诉国家"日志重建中, 不影响比赛"

## 12. 📖 词汇表

| 术语 | 含义 |
|---|---|
| **PO / 采购单** | Purchase Order, 生产前提 |
| **RECIPES** | 配方表 (前后端硬编码, 不在 Sheet) |

| **dice (运输骰 / 制造骰)** | 实体骰子, 国家亲手丢, 导师按按钮记录 |
